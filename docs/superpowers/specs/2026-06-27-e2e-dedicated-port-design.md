# E2E 専用ポート設計

## 背景

Playwright のローカル実行では、`reuseExistingServer` が有効なため、通常の開発サーバーが使用する `localhost:4321` を再利用することがある。Astro 開発サーバーを再利用すると Dev Toolbar がページに注入され、フッターリンクのクリックを妨げる。

## 採用方針

Playwright E2E テスト専用の固定ポートとして `4322` を使用する。

- `use.baseURL` を `http://localhost:4322/` に変更する。
- `webServer.url` を `http://localhost:4322/` に変更する。
- `webServer.command` のプレビュー起動時に `--port 4322` を渡す。
- `reuseExistingServer` の既存方針は維持する。通常の開発サーバーは `4321` のため、意図しない再利用は発生しない。
- `docs/e2e-testing.md` の説明を専用ポートに合わせる。

## 検討した代替案

### 環境変数でポートを設定する

柔軟性は高いが、現状は実行環境ごとにポートを変える要件がなく、設定箇所と運用上の分岐が増えるため採用しない。

### `reuseExistingServer` を常に無効にする

意図しない開発サーバーの再利用は防げるが、ポート競合そのものは解消しない。通常開発と E2E を同時に実行できる専用ポートの方が目的に合う。

## 検証

1. 変更前に、設定が `4322` を使用していないことを確認する。
2. 変更後に Playwright の設定一覧を読み込み、Web サーバーが `4322` で起動することを確認する。
3. `pnpm run test:e2e` を実行し、通常の開発サーバーを起動した状態でも E2E が独立して完走することを確認する。
4. `pnpm run lint` を実行する。
